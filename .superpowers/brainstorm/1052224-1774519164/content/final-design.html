n8n Queue Mode Flow

lw-s1: n8n main (UI + webhooks + API)
  → enqueues jobs to Redis (10.0.1.2:6379)
  → reads/writes state to Postgres (10.0.1.2:5432)

lw-c1: n8n-workers Deployment (2 replicas)
  → polls Redis queue for jobs
  → executes workflows
  → writes results to Postgres
  → env vars from K8s Secret (Ansible populates from Vault)

GitHub Runners on K8s

Simple approach: myoung34/github-runner as a K8s Deployment (2 replicas) on lw-c1. Same image as lw-s1 today, targets same repo. Shell executor inside pods. Registration token from Vault → K8s Secret.

Actions Runner Controller (ARC) left as a future upgrade path if job isolation becomes a requirement.

vCluster (test environment)

Deployed via Helm into test namespace on the K3s cluster. Full isolated K8s API surface. Ephemeral — tear down and recreate per test suite. ArgoCD can manage its lifecycle.
