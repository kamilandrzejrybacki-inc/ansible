VM Network

K8s VMs bridge to vmbr0 (physical LAN NIC) and get static IPs on 192.168.0.0/24 — same LAN as all other nodes. This avoids NAT complexity and lets workers reach lw-nas via the existing routing on lw-main.

n8n Queue Mode

k8s-w1 (K8s Deployment): n8n worker containers with same env vars, N8N_WORKER_MODE=true. Vault shim stays on lw-s1 — workers inherit env from K8s Secrets (populated by Ansible from Vault).

GitHub Actions Runners

Two options — decide before implementation:

Simple: myoung34 Deployment
Same image as lw-s1, runs as K8s Deployment on k8s-w2. Shell executor. Easy to manage via Ansible.
K8s-native: Actions Runner Controller
Operator that spins up ephemeral pods per workflow job. More complex, but each job is fully isolated.

vCluster (test environment)

Deployed via Helm into a test namespace on the main cluster. Provides a full isolated K8s API for integration/e2e tests. Ephemeral — can be torn down and recreated per test suite run via ArgoCD or Ansible.
