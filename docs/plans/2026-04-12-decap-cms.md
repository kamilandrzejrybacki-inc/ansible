# Decap CMS Deploy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy a self-hosted Decap CMS OAuth proxy on `lw-main` and render static admin files into the blog repo, enabling browser-based Markdown editing of `kamilrybacki.github.io` articles.

**Architecture:** Two Ansible roles — `oauth-proxy` (deploys `vencax/netlify-cms-github-oauth-provider` as a Docker container on `lw-main` + adds a Caddy vhost block without `import authelia`) and `decap-admin` (renders `/admin/index.html` and `/admin/config.yml` into a locally cloned blog repo). Secrets come from Vault at `secret/homelab/decap-cms`. Everything follows the existing homelab patterns: `community.docker.docker_container`, `blockinfile` on `/opt/homelab/caddy/Caddyfile`, Pi-hole DNS record, and `docker exec caddy caddy reload`.

**Tech Stack:** Ansible, community.docker collection, Jinja2 templates, Vault KV v2, Caddy (Docker), Pi-hole (Docker), `vencax/netlify-cms-github-oauth-provider` Docker image, Decap CMS CDN.

**Spec:** `docs/specs/2026-04-12-decap-cms-deploy-design.md`

These tasks assert the desired end state. They must **fail** on a clean host with no deployment. Run them after each subsequent task to confirm progress.

```yaml
# roles/oauth-proxy/tasks/verify.yml
---
- name: Gather oauth-proxy container info
  community.docker.docker_container_info:
    name: "{{ oauth_proxy_container_name }}"
  register: _oauth_container_info

- name: Assert oauth-proxy container is running
  ansible.builtin.assert:
    that:
      - _oauth_container_info.exists
      - _oauth_container_info.container.State.Running
    fail_msg: "oauth-proxy container is not running"
    success_msg: "oauth-proxy container is running"

- name: Check oauth-proxy responds on localhost
  ansible.builtin.uri:
    url: "http://127.0.0.1:{{ oauth_proxy_port }}/"
    status_code: 200
    timeout: 5
  register: _oauth_health

- name: Assert oauth-proxy health check passed
  ansible.builtin.assert:
    that: _oauth_health.status == 200
    fail_msg: "oauth-proxy did not return 200 on port {{ oauth_proxy_port }}"

- name: Check Caddy is serving the CMS auth vhost (any non-502 = proxy reachable)
  ansible.builtin.uri:
    url: "http://{{ saved_domain }}/callback"
    headers:
      Host: "{{ oauth_proxy_hostname }}.{{ saved_domain }}"
    status_code: [200, 302, 400, 404]
    timeout: 10
    validate_certs: false
  register: _caddy_vhost_check

- name: Assert Caddy vhost is not returning 502
  ansible.builtin.assert:
    that: _caddy_vhost_check.status != 502
    fail_msg: "Caddy returned 502 — proxy likely not reachable from Caddy container"
```

## Task 4: `oauth-proxy` deploy tasks

Reads secrets from Vault, creates the Docker container. Follows the pattern in `dev-tools/argocd-mcp-setup/roles/argocd-mcp/tasks/main.yml`.

**Files:** `roles/oauth-proxy/tasks/deploy.yml`

- [ ] **Step 1: Write deploy tasks**

```yaml
# roles/oauth-proxy/tasks/deploy.yml
---
- name: Ensure oauth-proxy state directory exists
  ansible.builtin.file:
    path: "{{ oauth_proxy_state_dir }}"
    state: directory
    mode: "0750"

- name: Read Decap CMS secrets from Vault
  ansible.builtin.uri:
    url: "{{ _vault_addr }}/v1/{{ _vault_kv_mount }}/data/homelab/decap-cms"
    headers:
      X-Vault-Token: "{{ _vault_token }}"
    return_content: true
    status_code: [200]
    timeout: 10
  register: _vault_decap
  no_log: true
  when: _vault_available | default(false) | bool

- name: Assert required Vault secrets are present
  ansible.builtin.assert:
    that:
      - _vault_decap.json.data.data.github_client_id is defined
      - _vault_decap.json.data.data.github_client_secret is defined
      - _vault_decap.json.data.data.cms_auth_origin is defined
    fail_msg: "Missing required keys in secret/homelab/decap-cms — expected: github_client_id, github_client_secret, cms_auth_origin"
  when: _vault_available | default(false) | bool

- name: Pull oauth-proxy Docker image
  community.docker.docker_image:
    name: "{{ oauth_proxy_image }}:{{ oauth_proxy_image_tag }}"
    source: pull
    force_source: "{{ oauth_proxy_image_tag == 'latest' }}"

- name: Record current image digest for rollback
  ansible.builtin.shell: |
    docker inspect --format='{{ '{{' }}.Id{{ '}}' }}' {{ oauth_proxy_image }}:{{ oauth_proxy_image_tag }}
  register: _oauth_image_digest
  changed_when: false

- name: Save image digest to state dir (rollback reference)
  ansible.builtin.copy:
    content: "{{ _oauth_image_digest.stdout }}\n"
    dest: "{{ oauth_proxy_state_dir }}/current_image_digest"
    mode: "0640"

- name: Deploy oauth-proxy container
  community.docker.docker_container:
    name: "{{ oauth_proxy_container_name }}"
    image: "{{ oauth_proxy_image }}:{{ oauth_proxy_image_tag }}"
    state: started
    restart_policy: "{{ oauth_proxy_restart_policy }}"
    ports:
      - "{{ ansible_default_ipv4.address }}:{{ oauth_proxy_port }}:3000"
    env:
      GITHUB_CLIENT_ID: "{{ _vault_decap.json.data.data.github_client_id }}"
      GITHUB_CLIENT_SECRET: "{{ _vault_decap.json.data.data.github_client_secret }}"
      ORIGIN: "{{ _vault_decap.json.data.data.cms_auth_origin }}"
    healthcheck:
      test: ["CMD-SHELL", "wget -qO- http://localhost:3000/ || exit 1"]
      interval: 30000000000
      timeout: 5000000000
      retries: 3
  no_log: true
```

## Task 5: `oauth-proxy` Caddy vhost tasks

Adds a vhost block to the homelab Caddyfile using `blockinfile` (exactly like all other services). No `import authelia` — this service IS the auth layer.

**Files:** `roles/oauth-proxy/tasks/caddy.yml`

- [ ] **Step 1: Write Caddy tasks**

```yaml
# roles/oauth-proxy/tasks/caddy.yml
---
- name: Add Decap CMS auth vhost to Caddyfile
  ansible.builtin.blockinfile:
    path: /opt/homelab/caddy/Caddyfile
    marker: "# {mark} DECAP_CMS_AUTH"
    unsafe_writes: true
    block: |
      http://{{ oauth_proxy_hostname }}.{{ saved_domain }} {
        import rate_limit
        import proxy_headers
        reverse_proxy http://{{ ansible_default_ipv4.address }}:{{ oauth_proxy_port }}
      }
  notify: reload caddy

- name: Add Pi-hole local DNS record for Decap CMS auth
  ansible.builtin.command: >
    docker exec pihole sh -c
    'grep -qF "{{ oauth_proxy_hostname }}.{{ saved_domain }}" /etc/pihole/custom.list ||
     (echo "{{ ansible_default_ipv4.address }} {{ oauth_proxy_hostname }}.{{ saved_domain }}" >> /etc/pihole/custom.list && echo CHANGED)'
  register: _pihole_dns_result
  changed_when: "'CHANGED' in _pihole_dns_result.stdout"
```

- [ ] **Step 2: Add the `reload caddy` handler to `main.yml` (handlers section)**

We add it as a handler at the role level. Ansible roles support `handlers/main.yml`. Create that file:

- [ ] **Step 2: Create `rollback.yml`**

```yaml
# roles/oauth-proxy/tasks/rollback.yml
---
- name: Read Decap CMS secrets from Vault (needed to restart container with env vars)
  ansible.builtin.uri:
    url: "{{ _vault_addr }}/v1/{{ _vault_kv_mount }}/data/homelab/decap-cms"
    headers:
      X-Vault-Token: "{{ _vault_token }}"
    return_content: true
    status_code: [200]
    timeout: 10
  register: _vault_decap
  no_log: true
  when: _vault_available | default(false) | bool

- name: Read previous image digest from state dir
  ansible.builtin.slurp:
    src: "{{ oauth_proxy_state_dir }}/current_image_digest"
  register: _prev_digest_raw

- name: Set previous image digest fact
  ansible.builtin.set_fact:
    _oauth_prev_digest: "{{ _prev_digest_raw.content | b64decode | trim }}"

- name: Stop current oauth-proxy container
  community.docker.docker_container:
    name: "{{ oauth_proxy_container_name }}"
    state: stopped

- name: Restart oauth-proxy from previous image digest
  community.docker.docker_container:
    name: "{{ oauth_proxy_container_name }}"
    image: "{{ _oauth_prev_digest }}"
    state: started
    restart_policy: "{{ oauth_proxy_restart_policy }}"
    ports:
      - "{{ ansible_default_ipv4.address }}:{{ oauth_proxy_port }}:3000"
    env:
      GITHUB_CLIENT_ID: "{{ _vault_decap.json.data.data.github_client_id }}"
      GITHUB_CLIENT_SECRET: "{{ _vault_decap.json.data.data.github_client_secret }}"
      ORIGIN: "{{ _vault_decap.json.data.data.cms_auth_origin }}"
  no_log: true
```

## Task 12: README

**Files:** `home-services/decap-cms/README.md`

- [ ] **Step 1: Write README**

```markdown
# decap-cms

Self-hosted Decap CMS for kamilrybacki.github.io. Provides a browser-based Markdown editor
at `https://kamilrybacki.github.io/admin/` backed by a GitHub OAuth proxy on `lw-main`.

## Prerequisites

- WireGuard VPN connected (lw-main is VPN-gated)
- Vault running and `~/.vault-ansible.yml` configured
- Blog repo cloned locally (for `make admin`)

## Setup

### 1. Create GitHub OAuth App

1. Go to: https://github.com/settings/developers → OAuth Apps → New OAuth App
2. Set:
   - **Application name:** Homelab Blog CMS
   - **Homepage URL:** `https://kamilrybacki.github.io`
   - **Authorization callback URL:** `http://cms-auth.<your-domain>/callback`
3. Copy the **Client ID** and generate a **Client Secret**

### 2. Store secrets in Vault

```bash
vault kv put secret/homelab/decap-cms \
  github_client_id="<client-id>" \
  github_client_secret="<client-secret>" \
  cms_auth_origin="https://kamilrybacki.github.io"
```

### 3. Clone blog repo (if not already)

```bash
git clone https://github.com/kamilrybacki/kamilrybacki.github.io /opt/data/kamilrybacki.github.io
```

### 4. Deploy

```bash
cd home-services/decap-cms
make deploy
```

## Runbook

**Container not starting:**
```bash
docker logs decap-cms-oauth
# Common: missing env var → check vault kv get secret/homelab/decap-cms
```

**OAuth login redirects to Authelia instead of GitHub:**
- The Caddyfile block for `cms-auth.<domain>` must NOT contain `import authelia`
- Check: `grep -A10 "DECAP_CMS_AUTH" /opt/homelab/caddy/Caddyfile`

**Caddy returns 502:**
```bash
docker inspect decap-cms-oauth --format '{{.State.Running}}'
# If false: make deploy
# If true: check port binding: docker port decap-cms-oauth
```

**`make rollback` fails with "file not found":**
- The state file is created on first deploy. If it's missing, pull a known-good image tag manually:
```bash
docker pull vencax/netlify-cms-github-oauth-provider:<tag>
# Then set oauth_proxy_image_tag in defaults/main.yml and run make update
```

**Admin files not showing in CMS:**
- Confirm `admin/config.yml` and `admin/index.html` are committed and pushed to GitHub
- GitHub Pages rebuild takes ~60s after push

## Required Vault Variables

| Path | Key | Description |
|------|-----|-------------|
| `secret/homelab/decap-cms` | `github_client_id` | GitHub OAuth App client ID |
| `secret/homelab/decap-cms` | `github_client_secret` | GitHub OAuth App client secret |
| `secret/homelab/decap-cms` | `cms_auth_origin` | CMS origin, e.g. `https://kamilrybacki.github.io` |

## Architecture Notes

- OAuth proxy port is bound to `lw-main`'s host IP only — not reachable without VPN
- Caddy vhost has no `import authelia` (by design — this service IS the auth provider)
- Admin panel is served by GitHub Pages — zero infra for the frontend
- Once logged in, Decap CMS talks directly to the GitHub API — proxy is idle
- Rollback uses image digest saved at deploy time (`/opt/decap-cms/current_image_digest`)
